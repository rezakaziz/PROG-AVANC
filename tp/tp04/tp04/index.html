    TP 04 : Introduction aux Design Patterns

    ⬇️ Télécharger le TP (tp04.zip)

    Objectif du TP

    L’objectif de ce TP est de découvrir et de pratiquer les design patterns structurels à travers des exemples simples et concrets.
    Vous allez ici manipuler deux patterns :

    • les décorateur
    • les composites

    💡 Ce TP met l’accent sur la réutilisabilité, la composition d’objets et la séparation des responsabilités.

    Instructions préliminaires

    Avant de commencer ce TP, vous devez :

    1. Créer un nouveau dépôt GitHub nommé tp04-patterns.
    2. Ajouter le compte @rezakaziz comme collaborateur du projet.
    3. Clonez votre dépôt en local et travaillez dessus pour les exercices suivants.
    4. Commitez et poussez régulièrement vos modifications : 
      git add .
git commit -m "progression"
git push

    Note

    Les tests unitaires sont à écrire par l'étudiant.

    Exercice 1 : Le pattern Décorateur

    Contexte

    Ce premier exercice s’inspire de la bibliothèque itertools de Python.
    L’objectif est de travailler sur des séquences de chaînes de caractères, en construisant progressivement une hiérarchie d’objets qui permettent de décorer une séquence existante avec des comportements supplémentaires.

    On suppose qu’on dispose d’une interface Sequence définissant une méthode : 

    String ligneSuivante();
    ainsi qu’une implémentation de base, ArraySequence, qui s’appuie sur un tableau de String.

    Question 1 — Décorateur UpperCase

    Écrivez et testez une classe UpperCaseDecoratorSequence, qui sera un décorateur de Sequence.
    Elle renverra les chaînes de caractères en majuscules.

    💡 Rappel : un décorateur doit implémenter la même interface que l’objet qu’il décore, et déléguer une partie du comportement à l’objet décoré.

    ✅ Exemple : 

    Sequence s = new UpperCaseDecoratorSequence(new ArraySequence(lignes));
System.out.println(s.ligneSuivante());

    Question 2 — Décorateur Grep

    Écrivez et testez une classe GrepSequence, qui sera un décorateur filtrant les lignes d’une autre séquence selon un texte donné.

    Exemple d’utilisation : 

    GrepSequence s = new GrepSequence(autreSequence, "chemin");
String l;
while ((l = s.ligneSuivante()) != null) {
    System.out.println(l);
}

    affiche uniquement les lignes de autreSequence contenant le mot "chemin".

    💭 Pistes de réflexion - Pourquoi un décorateur est-il plus flexible qu’un héritage direct ? - Que se passerait-il si on voulait combiner UpperCaseDecoratorSequence et GrepSequence ?

    Question 3 — Décorateur LineNumber

    Écrivez et testez une classe LineNumberSequence, qui sera un décorateur de Sequence maintenant en mémoire le numéro de la dernière ligne lue.

    Exemple : 

    LineNumberSequence s = new LineNumberSequence(autreSequence);
String l;
while ((l = s.ligneSuivante()) != null) {
    System.out.println(s.getLineNumber() + ": " + l);
}

    💡 Idée clé : - Vous pouvez stocker le compteur de lignes dans une variable d’instance incrémentée à chaque lecture.

    Question 4 — Décorateur Slice

    Créez une classe SliceSequence, qui sera un décorateur renvoyant seulement une partie des lignes de la séquence d’origine.
    Le constructeur prendra deux entiers : - start : indice de la première ligne incluse - end : indice de la première ligne à ignorer.

    Exemple : 

    SliceSequence s = new SliceSequence(autreSequence, 10, 20);
String l;
while ((l = s.ligneSuivante()) != null) {
    System.out.println(l);
}
    Cet exemple affiche les lignes de 10 (incluse) à 20 (exclue).

    💡 Astuce : Réutilisez votre implémentation de la LineNumberSequence pour simplifier la gestion des indices !

    À retenir

    Le pattern Décorateur permet :

    • d’ajouter dynamiquement des responsabilités à un objet ;
    • de composer des comportements sans explosion de sous-classes ;
    • et de respecter le principe ouvert/fermé (Open/Closed Principle de SOLID*).

    Exercice 2 : Le pattern Composite

    Contexte

    Cet exercice s’appuie sur la représentation d’un document XML sous forme d’arbre d’objets.
    Chaque nœud du document (élément, texte, attribut…) est un objet d’une hiérarchie commune.
    Le but est de générer du XML à partir de cette structure, selon le pattern Composite.

    Question 1 — Génération XML

    On vous fournit une implémentation du composite XML et un exemple montrant comment extraire le texte contenu dans tout le document.

    Vous devez implémenter la méthode : 

    String toXML();
    pour les différentes classes du composite.

    💡 Règle d’affichage :

    • Les attributs doivent être listés par ordre alphabétique de leur nom (pour un comportement déterministe).
    • Vous pouvez ignorer les différences entre balises vides <img/> et <img></img>.

    Question 2 — Réflexion sur les patterns utilisés

    💭 Analysez votre code :

    • Quels avantages concrets vous apportent les patterns Décorateur et Composite ?
    • Quels problèmes d’évolution ou de maintenance auraient pu apparaître sans eux ?
